<h2>Nil значения</h2>

<h3>Теория:</h3>

<ul>
    <li>
        Сборщик мусора нужен для того, чтобы освобождать память в куче, от данных, который больше не нужны (на которые нет валидных ссылок).
    </li>
    <li>
        Механизм. При выполнение функции foo в стеке переменная, например, создается в куче. Как только функция foo отработает, то ссылка на нее будет невалидной (невалидная память). Когда новая функция bar перезатрет старую foo, так как она полностью отработала, то переменная будет удалена.
    </li>
    <li>
        Маркировка.
        В куче есть объекты (помечены белым).
        Берутся ссылки из стека и кладутся в очередь (помечаются серым).
        Проверяется ссылка из очереди. Если если она указывает на другие ссылки, то считается, что она живая (помечается черным). Ссылки, которые были найдены добавляются в очередь (помечаются серым).
        Выполняется следующая итерация по всем ссылкам.
        При завершение всех итераций GC выключается, освобождается память по тем ссылкам, на которые больше не ссылаются (помечаются красным).
        Аллоцируются новые ссылки (помечаются белыми).
        Происходит новая итерация GC. 
    </li>
    <li>
        Барьер на запись. Так как GC работает параллельно с основной программой, то для аллокации новых объектов применяется Барьер на запись. Он помечает их живыми (помечаются черными). Это позволяет GC при терации игнорировать новые переменные.
    </li>
</ul>
